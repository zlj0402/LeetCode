# M-Bus 协议全面总结

## 一、协议体系概述

### EN 13757 标准系列

| 标准 | 内容 |
|------|------|
| EN 13757-1 | 数据交换 - 通用描述和通信协议 |
| EN 13757-2 | 有线 M-Bus 通信 - 物理层和链路层 |

### 协议层次模型

```
┌─────────────────────────────────────┐
│        应用层 (APL)                  │  EN 13757-3
│   M-Bus 应用协议 / DLMS/COSEM        │
├─────────────────────────────────────┤
│    认证和分段子层 (AFL) - 可选        │  EN 13757-7
│   分段、MAC、消息计数器               │
├─────────────────────────────────────┤
│        传输层 (TPL)                  │  EN 13757-7
│   帧 header: 短/长/无                │
│   地址、配置字段、安全模式            │
├─────────────────────────────────────┤
│      扩展链路层 (ELL) - 可选         │  EN 13757-4
│   同步传输、时间窗口                  │
│   **仅无线M-Bus**                    │
├─────────────────────────────────────┤
│        链路层 (DLL)                  │  EN 13757-2/4
│   有线: FT1.2 / 无线: FT3            │
├─────────────────────────────────────┤
│        物理层 (PHY)                  │  EN 13757-2/4
│   有线: 双绞线 / 无线: SRD 频段       │
└─────────────────────────────────────┘
```

### 有线 vs 无线协议层次差异

| 层次 | 有线 M-Bus | 无线 M-Bus |
|------|------|------|
| ELL | **不支持** | 支持（同步传输） |
| DLL帧格式 | FT1.2 | FT3 |
| 地址携带 | 链路层仅主地址 | 链路层完整8字节地址 |
| TPL Header | 必须长Header | 可用短Header |
| 仪表主动发送 | **不支持** | 支持 |

**ELL (Extended Link Layer) 说明**：
- 定义于 EN 13757-4
- 仅用于无线 M-Bus
- 提供同步传输和时间窗口管理
- 有线 M-Bus 不需要 ELL，因为：
  - 有线连接是持续稳定的
  - 主站完全控制通信时序
  - 无需无线特有的同步机制

## 二、链路层 (DLL)

### 有线 M-Bus (EN 13757-2)

**帧结构 (FT1.2)**：
```
┌─────┬─────┬─────┬─────┬─────┬─────┐
│Start│ L   │ C   │ A   │Data │CS   │End  │
│ 10h │     │     │主地址│     │     │16h  │
└─────┴─────┴─────┴─────┴─────┴─────┘
- 波特率: 2400 b/s, E,8,1
- 地址: 1字节主地址 + 1字节次地址
- 不包含完整仪表地址信息（制造商、识别号等）
```

**地址结构**：
- 主地址: 0-250（设备编号），255 = 广播/通配符
- 次地址: 用于选择功能

### 无线 M-Bus (EN 13757-4)

**帧结构 (FT3)**：
```
┌─────┬─────┬─────┬─────┬─────┬─────┐
│Preamble│L   │C   │Addr │Data │CRC  │
│        │     │     │8字节│     │     │
└─────┴─────┴─────┴─────┴─────┴─────┘
- 地址包含完整信息: 制造商(2B) + 识别号(4B) + 版本(1B) + 设备类型(1B)
```

**传输模式**：
| 模式 | 方向 | 频率 | 特点 |
|------|------|------|------|
| C1 | 单向 | 868 MHz | 紧凑格式，快速传输 |
| C2 | 双向 | 868 MHz | 支持确认和命令 |
| T1 | 单向 | 868 MHz | 长格式，更多数据 |
| T2 | 双向 | 868 MHz | 支持确认和命令 |
| S1/S1-m | 单向 | 868 MHz | 慢速模式，节能 |
| R2 | 双向 | 868 MHz | 中继模式 |

## 三、传输层 (TPL)

### TPL Header 类型

| 类型 | 长度 | 使用场景 |
|------|------|------|
| **无 Header** | 0 字节 | 简单传输，无安全需求 |
| **短 Header** | 4 字节 | 无线 M-Bus，仪表直接通信 |
| **长 Header** | 12 字节 | 有线 M-Bus / 网关场景 |

### 短 TPL Header 结构 (表11)

```
┌─────────┬─────────┬───────────────────┬──────────────┐
│访问号   │状态     │配置字段 (CF)       │CF扩展 (可选) │
│ 1字节   │ 1字节   │ 2字节              │ 0-n字节      │
└─────────┴─────────┴───────────────────┴──────────────┘

适用条件：
1. 无线 M-Bus 系统（链路层已包含完整地址）
2. 仪表地址与链路层地址相同（仪表直接通信）
3. 不经过网关/转换器
```

### 长 TPL Header 结构 (表12)

```
┌─────────┬─────────┬─────┬─────┬─────┬─────┬─────┬─────┐
│识别号   │制造商   │版本 │设备 │访问 │状态 │CF   │CFE  │
│ 4字节   │ 2字节   │1字节│1字节│1字节│1字节│2字节│0-n │
└─────────┴─────────┴─────┴─────┴─────┴─────┴─────┴─────┘

适用场景：
1. 有线 M-Bus（链路层无完整地址）
2. 网关/转换器代理通信（仪表地址 ≠ 链路层地址）
3. 需要明确标识数据归属仪表
```

### CI 字段 (控制信息)

| CI 值 | 含义 |
|------|------|
| 7Ah | 短 TPL header，无加密 |
| 72h | 长 TPL header，无加密 |
| 5Bh | 长 TPL header，应用层响应 |
| 8Ah | 短 TPL header，安全模式 5 |
| ... | 其他安全模式对应值 |

## 四、配置字段 (Configuration Field - CF)

### CF 基本结构 (7.5.8)

配置字段位于 TPL Header 中，长度 2 字节（可扩展），定义数据报的处理方式。

### CF 子字段定义 (7.7.1)

| 子字段 | 位位置 | 描述 |
|------|------|------|
| **H/R** | 位15 | Header类型：0=短Header，1=长Header |
| **CC** | 位14-13 | 通信控制：00=单向，01=双向响应，10=双向命令 |
| **IIII** | 位12-9 | 访问号增量（0-15），指示访问号变化量 |
| **B/A/S** | 位8 | 分段/认证/安全模式指示 |
| **KKKK** | 位7-4 | 密钥ID（0-15），标识使用的密钥 |
| **V** | 位3 | 填充指示：0=无填充，1=有填充 |
| **DD** | 位2-0 | 填充长度（0-7字节），V=1时有效 |

### CF 扩展字段 (CFE)

当基本 CF 不能满足需求时，使用扩展字段：
- **基本 CF**: 2字节，适用于简单场景
- **扩展 CF**: 基本CF + 扩展字节，用于复杂安全配置

### 安全模式对应的 CF 结构 (7.7.2-7.7.5)

**模式 5 (AES-CBC-128)**：
```
CF: H/R | CC | IIII | B/A/S=1 | KKKK | V | DD
扩展字节: 密钥版本等
```

**模式 7 (AES-CBC + CMAC)**：
```
CF: H/R | CC | IIII | B/A/S=1 | KKKK | V | DD
需要 AFL 支持认证
```

**模式 9 (AES-GCM)**：
```
CF: H/R | CC | IIII | B/A/S=1 | KKKK | V=0 | DD=0
单次认证加密，与 DLMS/COSEM 兼容
```

**模式 10 (AES-CCM)**：
```
CF: H/R | CC | IIII | B/A/S=1 | KKKK | V | DD
最小开销优化
```

### CF 依赖结构 (7.6)

不同安全模式下，CF 决定后续数据结构：
| 安全模式 | CF 后续结构 |
|------|------|
| 0 | 无加密，直接 APL 数据 |
| 5 | 加密数据块（AES-CBC） |
| 7 | AFL Header + 加密数据 + MAC |
| 8 | 加密数据 + 双 MAC |
| 9 | 加密数据 + MAC（GCM tag） |
| 10 | 加密数据 + MAC（CCM tag） |

## 五、地址系统

### 识别号 (Identification Number)

- 编码: 8 位 BCD 紧缩数字 (4 字节)
- 范围: 00000001 ~ 99999999
- 类型:
  - **制造编号**: 出厂预设，唯一，通常不可更改
  - **可更改编号**: 可按系统集成需求修改（需另有固定制造编号）
- **通配符**: Fh 位置可替换 0-9，用于组寻址

示例: `112233FF` 匹配 `11223300` ~ `11223399`

### 制造商 ID (Manufacturer ID)

- 编码: 2 字节无符号二进制
- 计算公式: `[ASCII(1st)-64]×1024 + [ASCII(2nd)-64]×32 + [ASCII(3rd)-64]`
- 管理: FLAG 协会 (http://www.dlms.com/flag)
- **通配符**: FFFFh 匹配所有制造商

### 版本 (Version)

- 编码: 1 字节
- 用途: 设备代数/版本标识
- **通配符**: FFh 匹配所有版本

## 六、访问号 (Access Number)

### 功能

1. **检测新数据**: 通过访问号递增判断数据报是否为新数据
2. **检测重复**: 相同访问号的数据报被视为重复，应忽略
3. **检测频繁读出**: (有线M-Bus) 用户可通过访问号变化频率发现异常抄表

### 有线 vs 无线访问号规则 (7.5.5.2/7.5.5.3)

**无线 M-Bus**：
| 方向 | 规则 |
|------|------|
| 仪表主动发送 | 每次递增 CM (模256)，响应时重复伙伴访问号 |
| 伙伴发送 | 300s内使用不同值，响应时重复仪表访问号 |
| 访问号用途 | 配对请求-响应，判断新数据 |

**有线 M-Bus**：
| 方向 | 规则 |
|------|------|
| 仪表主动发送 | **不允许**（有线不支持仪表发起） |
| 仪表响应 | **使用并递增自己的访问号**，可忽略主站发送的访问号 |
| Master 发送 | 普通命令访问号可忽略，加密命令必须提供 |
| 访问号用途 | "被读次数"计数，供用户检测异常读取 |

**有线 vs 无线对比**：
| 方面 | 无线 M-Bus | 有线 M-Bus |
|------|------|------|
| 仪表响应 | 重复伙伴访问号 | 递增自己的访问号 |
| Master 访问号 | 必需（配对） | 可被仪表忽略 |
| 仪表主动发送 | 允许 | **不允许** |

### 网关 CCP 偏移量策略

网关设置 CCP 时添加偏移量 (CCP = CM + 偏移量)，原因：
- **预留计数器空间**: 避免网关发送前仪表更新 CM 导致命令被丢弃
- **容错延迟**: 网关有足够时间准备和发送命令
- **偏移量范围**: 1-100（与 CCP 增量限制匹配）

示例：
```
仪表 CM = 350 → 网关设置 CCP = 400（偏移50）
仪表后续发送 CM = 351, 352, 353...
网关发送命令 CCP = 400
仪表验证：400 在 [354, 453] 范围内 → 接受
若无偏移(CCP=350) → 350 < 354 → 丢弃（被认为过期）
```

### 检测规则

```
接收方处理：
- 访问号 = 上次接收值 → 重复数据报 → 忽略
- 访问号 ≠ 上次接收值 → 新数据报 → 处理

无线场景（严格检测）：相同访问号 = 重复 = 忽略
有线场景（宽松检测）：Master 可选择检查仪表访问号变化
```

## 七、状态字节 (Status Byte)

### 仪表消息状态 (表14-15)

| 位 | 含义(置位) | 含义(未置位) |
|------|------|------|
| 7-5 | 制造商特定 | 制造商特定 |
| 4 | 暂时错误 | 无暂时错误 |
| 3 | 永久错误 | 无永久错误 |
| 2 | 电量低 | 电量正常 |
| 1-0 | 应用状态 | 见表15 |

**应用状态编码 (位1-0)**：
| 值 | 含义 |
|------|------|
| 00 | 无错误 |
| 01 | 应用忙 |
| 10 | 应用错误 |
| 11 | 异常条件/告警 |

### 错误类型说明

- **电量低**: 外部电源中断或电池寿命结束
- **永久错误**: 致命设备错误，需要服务干预
- **暂时错误**: 轻微错误，可能自动恢复
- **应用错误**: 命令解释/执行失败
- **异常条件**: 正常工作时检测异常行为（如水表持续流动）

### 伙伴消息状态 (7.5.7)

伙伴发送的消息中，状态字节含义不同：
| 位 | 含义 |
|------|------|
| 7 | More fragments follow (MF) - 更多分段跟随 |
| 6-5 | Reserved - 保留 |
| 4-3 | Error type - 错误类型 |
| 2-0 | Reserved - 保留 |

**伙伴状态用途**: 主要用于分段传输控制，指示是否有更多分段。

## 八、认证和分段子层 (AFL)

### AFL 提供的服务

1. **分段**: 将长消息分为多个数据报传输
2. **认证**: MAC 保护 TPL 和 APL 真实性
3. **消息计数器**: 安全相关消息标识

### AFL 字段结构 (表3)

| 字段 | 长度 | 描述 |
|------|------|------|
| CI | 1B | 指示 AFL 紧随 |
| AFLL | 1B | AFL 长度（不含 AFLL） |
| FCL | 2B | 分段控制字段 |
| MCL | 1B | 消息控制字段 |
| KI | 2B | 密钥信息字段 |
| MCR | 4B | 消息计数器字段 |
| MAC | N B | 消息认证码 |
| ML | 2B | 消息长度字段 |

### 分段控制字段 FCL (EN 13757-7 Table 4)

| 位 | 字段 | 英文定义 | 描述 |
|------|------|----------|------|
| 15 | RES | Reserved | 保留 (0b) |
| 14 | MF | More-Fragments | 0b=最后段, 1b=有后续段 |
| 13 | MCLP | Message Control **in this Fragment** Present | MCL 在当前分段存在 |
| 12 | MLP | Message Length **in this Fragment** Present | ML 在当前分段存在 |
| 11 | MCRP | Message counter **in this Fragment** Present | MCR 在当前分段存在 |
| 10 | MACP | MAC **in this Fragment** Present | MAC 在当前分段存在 |
| 9 | KIP | Key Information **in this Fragment** Present | KI 在当前分段存在 |
| 8 | RES | Reserved | 保留 (0b) |
| 7-0 | FID | Fragment-ID | 分段ID (不分段=0, 分段首段=1并递增) |

**注意**: 0 = 字段不存在; 1 = 字段存在

### 消息控制字段 MCL (EN 13757-7 Table 5)

| 位 | 字段 | 英文定义 | 描述 |
|------|------|----------|------|
| 7 | RES | Reserved | 保留 (0b) |
| 6 | MLMP | Message Length **in Message** Present | 消息中消息长度存在 |
| 5 | MCMP | Message counter **in Message** Present | 消息中消息计数器存在 |
| 4 | KIMP | Key Information **in Message** Present | 消息中密钥信息存在 |
| 3-2 | AT | Authentication-Type | 认证类型 |

**重要说明** (EN 13757-7 6.3.3):
- "bits 4 to 7 define presence of additional fields **in the message**"
- MCL **只在第一分段存在** (由 FCL.MCLP=1 仅首段设置)
- 后续分段无 MCL，沿用首段定义

**注意**: 0 = 字段不存在; 1 = 字段存在

**认证类型 AT 编码**：
| 值 | 类型 | MAC 长度 |
|------|------|------|
| 0 | 无认证 | - |
| 1-2 | 保留 | - |
| 3 | AES-CMAC-128 | 2字节 |
| 4 | AES-CMAC-128 | 4字节 |
| 5 | AES-CMAC-128 | 8字节 |
| 6 | AES-CMAC-128 | 12字节 |
| 7 | AES-CMAC-128 | 16字节 |
| 8 | AES-GMAC-128 | 12字节 |
| 9-15 | 保留 | - |

### 密钥信息字段 KI (表7)

| 位 | 字段 | 描述 |
|------|------|------|
| 15-8 | 密钥版本 | 标识密钥版本 |
| 7-6 | RES | 保留 |
| 5-4 | KDF 选择 | 密钥派生函数选择 |
| 3-0 | 密钥 ID | 密钥标识 |

### AFL 字段加密状态 (EN 13757-7 6.3)

**重要**: AFL 层所有字段都是 **未加密** 的，但可被 MAC 认证保护。

| 字段 | 加密状态 | MAC 认证 |
|------|----------|----------|
| AFLL | 未加密 | 不被 MAC 保护 |
| FCL | 未加密 | 不被 MAC 保护 |
| MCL | 未加密 | 被 MAC 保护 |
| KI | 未加密 | 被 MAC 保护 |
| MCR | 未加密 | 被 MAC 保护 |
| MAC | 未加密 | MAC 值本身 |
| ML | 未加密 | 被 MAC 保护 |

**加密边界**: TPL 层的 "Encrypted application data" 才是加密部分。

### MAC 字段规则

- **分段 MAC**: 由 FCL.MACP 控制，可每个分段存在
- **消息 MAC**: 由 MCL.AT 定义，仅最后分段存在
- **传输顺序**: MSB 先传输（与其他多字节字段不同）
- **MAC 是 CMAC 计算结果**: MAC = CMAC(Kmac, 输入数据) 的输出值

### 分段时字段分布 (EN 13757-7 6.3.2/6.3.3)

**规则**: MCL、MCR、ML **只存在于第一分段**，后续分段不存在。

| 字段 | 第一分段 | 后续分段 | FCL控制位 |
|------|----------|----------|-----------|
| MCL | 存在 | 不存在 | MCLP (首段=1, 后续=0) |
| MCR | 存在 | 不存在 | MCRP (首段=1, 后续=0) |
| ML | 存在 | 不存在 | MLP (首段=1, 后续=0) |
| MAC | 可存在 | 可存在 | MACP (每段独立控制) |
| KI | 存在 | 不存在 | KIP (首段=1, 后续=0) |

**EN 13757-7 原文**: "If the AFL.MCL field is used it always shall be present in the first fragment. It shall not be present in any following fragments of the same message."

**分段结构示例**:
```
分段1 (FID=1): FCL(MCLP=1,MCRP=1,MACP=1) | MCL | MCR | ML | KI | 数据 | MAC
分段2 (FID=2): FCL(MCLP=0,MCRP=0,MACP=1) | 数据 | MAC
分段3 (FID=3): FCL(MCLP=0,MCRP=0,MACP=1) | 数据 | MAC | MAC_消息
```

## 九、消息计数器

### 计数器类型

| 计数器 | 用途 | 管理方 |
|------|------|------|
| CM | 仪表传输计数器 | 仪表 |
| CCP | 伙伴传输计数器 | 通信伙伴 |
| C'CP | 伙伴计数器未验证副本 | 仪表接收存储 |
| C'M | 仪表计数器未验证副本 | 伙伴接收存储 |
| C"M | 仪表计数器已验证副本 | 伙伴永久存储 |

### CM 规则 (仪表传输)

- 初始值: 0
- 每次生成认证/加密消息前递增 1
- **在未加密部分传输**（用于重放检测）
- 达到最大值回绕 → 建议在回绕前更换密钥
- 限制性系统不允许回绕 → 密钥更换时重置

### CCP 规则 (伙伴传输)

- 每仪表初始值: 0
- 生成认证/加密消息前递增
- **增量不超过 100**
- 可在收到仪表消息时设置
- 达到最大值回绕到 0-99

### 接收验证

**仪表接收伙伴消息**：
- 接收值不在 CM+1 ~ CM+100 范围 → 丢弃
- 有效范围 → 存为 C'CP
- 认证成功且 C'CP > CM → CM = C'CP

**伙伴接收仪表消息**：
- 接收值 ≤ C"M → 丢弃（重放）
- 接收值 > C"M → 存为 C'M
- 认证成功 → C"M = C'M

### 重放检测机制

```
接收方处理流程：
1. 读取未加密部分的消息计数器
2. 与本地已验证副本比较
3. ≤ 已验证值 → 旧消息重放 → 直接丢弃（无需解密）
4. > 已验证值 → 可能新消息 → 解密和 MAC 验证
5. 验证成功 → 更新已验证副本
```

### Z 字段与消息计数器 (EN 13757-7 Table 33)

**CF.Z (位13) - Counter size**：
- Z=0: counter size is 0 byte (not present in TPL)
- Z=1: counter size is 4 bytes (present in TPL)

当 Z=1 时，TPL 中有 **额外的 4 字节消息计数器字段**。

**消息计数器来源选择**：
| 来源 | 位置 | 长度 | 控制方式 |
|------|------|------|----------|
| TPL Counter Field | TPL 层 | 4字节 | CF.Z=1 |
| AFL.MCR | AFL 层 | 4字节 | FCL.MCRP=1 |

**注意**：两者是**不同位置的字段**，不是复用关系。

### 附录 B 消息计数器示例 (Annex B)

**示例场景**: 仪表与伙伴双向通信

```
初始状态：
- CM = 0（仪表计数器）
- CCP = 0（伙伴计数器）
- C"M = 0（伙伴已验证副本）

仪表发送第一条消息：
- CM 递增 → CM = 1
- 消息携带 CM = 1（未加密部分）

伙伴接收第一条消息：
- CM=1 > C"M=0 → 新消息
- 解密验证成功 → C"M = 1

伙伴发送第一条消息：
- CCP 递增 → CCP = 1
- 消息携带 CCP = 1

仪表接收第一条伙伴消息：
- CCP=1 在 CM+1~CM+100 范围内 → 有效
- 验证成功 → CM = 1（已同步）

后续通信示例：
- 仪表 CM = 5, 伙伴 CCP = 5
- 伙伴收到 CM=5 → C"M = 5
- 仪表收到 CCP=5 → CM = 5
- 伙伴发送 CCP=105（偏移100）
- 仪表验证：105 在 6~105 范围 → 有效
```

**重放攻击防护示例**：
```
攻击者截获 CM=10 的消息并重发：
伙伴接收：
- CM=10 ≤ C"M=10（假设已接收过）
- 直接丢弃，不消耗解密资源

攻击者修改计数器为 CM=11：
伙伴接收：
- CM=11 > C"M=10 → 可能新消息
- 解密 → MAC验证失败（攻击者无法伪造MAC）
- 丢弃，不更新 C"M
```

## 十、安全服务 (Chapter 9)

### 密钥类型概述

**M-Bus 使用对称密钥加密**：

- **算法**: AES-128（对称加密算法）
- **密钥长度**: 16字节（128位）
- **密钥共享**: 仪表与 HES/通信伙伴共享同一密钥
- **密钥来源**: 出厂预置或 SITP 安全传输
- **非对称密钥**: **不使用**（仪表资源受限，不适合公钥加密）

**EN 13757-7:2018 9.1 原文声明**：
> "安全服务基于对称密钥方法的使用。AES 算法及其操作模式已获得丰富经验，其强度已由密码专家评估。"

### 密钥生命周期

**密钥来源**：

| 阶段 | 来源 | 说明 |
|------|------|------|
| 出厂预置 | 制造商写入 | 每个仪表唯一密钥（至少8字节不同） |
| 安装/系统集成 | SITP 传输 | 通过现有密钥加密传输新密钥 |
| 运行更换 | SITP 更新 | 密钥更新、激活、停用、销毁 |

**密钥存储架构**：

```
仪表密钥存储
├── 持久密钥（长期存储）
│   ├── Key ID 0-15，每个可有多个版本
│   └── AES-128 密钥 (16字节)
│
└── 临时密钥（动态生成，不持久存储）
    ├── 从持久密钥通过 KDF 派生
    └── 用于单次消息加密/认证
```

**密钥数量关系**：

| 安全模式 | 密钥使用方式 | 密钥数量 |
|------|------|------|
| 模式 5 | 持久密钥直接使用 | 1个（加密） |
| 模式 7 | 持久密钥派生临时密钥 | 1个MK → 2个临时（加密+认证） |
| 模式 8 | 多个持久密钥 | 3个（Kenc₁, Kmac₁, Kmac₂） |
| 模式 9/10 | 持久密钥派生临时密钥 | 1个MK → 1个临时（加密+认证共用） |

**密钥管理流程**：

```
密钥更换流程 (SITP)：
1. HES → ACESM: ACTION.req(set_encryption_key) - 设置新密钥
2. HES → ACESM: ACTION.req(transfer_security_information) - SITP传输
3. ACESM → M-Bus: SND_UD2 (CI=C3h, SITP-Block BCF=00h) - 传输密钥
4. M-Bus → ACESM: RSP_UD (CI=C5h, SITP-Block BCF=80h) - 响应
5. ACESM → M-Bus: SND_UD2 (CI=C3h, SITP-Block BCF=01h) - 激活密钥
6. M-Bus → ACESM: RSP_UD (CI=C5h, SITP-Block BCF=81h) - 响应
7. ACESM → HES: GET.req(invocation_status) - 状态查询
```

### 安全目标 (表46)

| 安全服务 | 目标 |
|------|------|
| **完整性** | 防止未经授权方修改信息 |
| **真实性** | 确定通信方真实身份 |
| **机密性** | 防止向未经授权方披露信息 |

### 安全机制 (表47)

| 模式 | 算法 | 操作模式 | 加密 | 认证 | 开销 | MAC 长度 |
|------|------|------|------|------|------|------|
| 0 | 无 | - | - | - | 0 | 0 |
| 2 | DES | CBC | - | - | 1-8 | 0 |
| 3 | DES | CBC | - | - | 1-8 | 0 |
| 5 | AES-128 | CBC | ✓ | - | 2-17 | 0 |
| 7 | AES-128 | CBC | ✓ | CMAC | 3-23 | 2-16 |
| 8 | AES-128 | CTR | ✓ | CMAC | 6-16 | 4+2 |

### 模式说明

- **模式 0**: 无安全
- **模式 2/3**: DES-CBC，已弃用
- **模式 5**: 简单加密，单持久密钥，无密钥派生
- **模式 7**: 加密+AFL认证，支持密钥派生
- **模式 8**: 紧凑方案，双 MAC 端到端完整性
- **模式 9**: 单次认证加密，与 DLMS/COSEM 兼容
- **模式 10**: 认证加密，最小开销优化

### 安全模式 5 (AES-CBC-128)

**初始化向量 IV (表48)**：
```
15-14: 制造商 LSB
13-10: 识别号 LSB
9:     版本
8-5:   设备类型
4-0:   访问号（每次传输变化）
```

特点：
- 每次传输 IV 不同（访问号变化）
- 即使数据相同，加密结果也不同
- 防止零消耗检测和重放

**解密验证**: 前 2 字节为 2Fh (AES-check)

### 安全模式 7 (AES-CBC-128)

- IV = 0（静态）
- 使用临时密钥（通过 KDF 生成）
- 需要消息计数器（TPL 或 AFL.MCR）
- 数据填充至 16 字节倍数
- 解密验证: 前 2 字节 2Fh
- 认证: 使用 AFL 的 AES-CMAC (9.3.2)

**TPL 填充**: 1-16 字节，值等于填充字节数

### AFL 认证方法

**AES-CMAC-128 (9.3.2)**：
```
输入: AFL.MCL || {AFL.KI} || {AFL.MCR} || {AFL.ML} || TPDU
输出: 16字节 MAC，可截断为 4/8/12/16字节
```

**CMAC 截断规则**：
- 截断从 **MSB端（最高有效字节端）** 开始
- 截断为4字节: 取前4字节 (MAC[0-3])
- 截断为8字节: 取前8字节 (MAC[0-7])
- 截断为12字节: 取前12字节 (MAC[0-11])
- 截断越短，安全性越低，但开销越小

**AES-GMAC-128 (9.3.3)**：
```
IV = 固定字段(8B) + 调用字段(5B)
固定字段 = 制造商ID || 识别号 || 版本 || 设备类型
调用字段 = AFL.MCR || AFL.FID
输入: AFL.MCL || {AFL.KI} || {AFL.MCR} || {AFL.ML} || TPDU
输出: 12字节 MAC，不截断
```

### AES-CBC-128 加密机制详解

**基本参数**：
| 参数 | 值 | 描述 |
|------|------|------|
| 密钥长度 | 16字节 (128位) | AES-128 标准密钥 |
| 块大小 | 16字节 (128位) | AES 固定块大小 |
| IV 长度 | 16字节 | 必须与块大小相同 |

**CBC 链式加密过程**：
```
1. 第一个数据块:
   C[0] = AES_Encrypt(K, IV XOR P[0])

2. 后续数据块:
   C[i] = AES_Encrypt(K, C[i-1] XOR P[i])

3. 最终输出:
   加密数据 = C[0] || C[1] || ... || C[n-1]
```

**IV 的作用**：
- 使相同明文产生不同密文
- 防止攻击者识别数据模式
- 每次传输必须使用不同 IV（安全模式5通过访问号变化实现）
- IV 不需要保密，但必须不可预测或唯一

**解密过程**：
```
1. 第一个密文块:
   P[0] = AES_Decrypt(K, C[0]) XOR IV

2. 后续密文块:
   P[i] = AES_Decrypt(K, C[i]) XOR C[i-1]
```

**AES-check 验证**：
- 加密后数据前2字节应为 `2Fh`
- 解密后验证前2字节是否为 `2Fh`
- 验证失败 → 解密错误或密钥错误

| 24h | 数据内容错误: 密钥计数器错误 |
| 25h | 数据内容错误: 解封装失败 (ICV 检查失败) |
| 26h | 数据内容错误: MAC 错误 |
| 27h | 数据内容错误: 未知协议标识符 |

### 有线 M-Bus 安全模式 5 示例 (CEN/TR 17167 F.2)

**设备信息示例**：
| 参数 | 值 |
|------|------|
| 设备类型 | 热成本分配器 (HCA) - 08h |
| 制造商 | QDS (93h 44h) |
| 仪表识别号 | 55667788 |
| 版本 | 85 (55h) |
| AES 密钥 | 16字节对称密钥 |

**IV 初始化向量 (模式 5)**：
```
IV (16字节) = 制造商(2B) + 识别号(4B, LSB先) + 版本(1B) + 设备类型(1B) + 访问号(8B)
示例: 93 44 88 77 66 55 55 08 00 00 00 00 00 00 00 00
```

**有线 M-Bus RSP-UD 帧结构示例**：
```
帧结构 (FT1.2):
┌─────────┬────────────────────────────────────────────────┐
│ DLL     │ 68h 22h 22h 68h 08h FDh                        │
├─────────┼────────────────────────────────────────────────┤
│ TPL     │ CI=72h (长Header)                              │
│         │ 识别号: 88 77 66 55                            │
│         │ 制造商: 93 44                                  │
│         │ 版本: 55h                                      │
│         │ 设备类型: 08h                                  │
│         │ 访问号: 00h                                    │
│         │ 状态: 04h                                      │
│         │ CF: 00h 05h (Mode 5, 1个加密块)               │
├─────────┼────────────────────────────────────────────────┤
│ 加密块  │ AES-check: 2F 2F → 加密后: 00 DF              │
│ (16B)   │ 应用数据 (DR1+DR2+DR3) → 加密后: E2 A7...     │
├─────────┼────────────────────────────────────────────────┤
│ 明文    │ DR4: 温度 25°C (01 5B 19) - 不加密             │
├─────────┼────────────────────────────────────────────────┤
│ DLL尾   │ Checksum: 40h, Stop: 16h                      │
└─────────┴────────────────────────────────────────────────┘
```

**部分加密特性**：
- 加密块: AES-check(2B) + 应用数据(14B) = 16字节
- 明文部分: 温度数据(3B) - 允许未加密访问操作参数
- CF.NNNN = 0001b 表示 1个加密块

### 有线 M-Bus 安全模式 7 示例 (CEN/TR 17167 F.3 Table F.5)

**设备信息**：
| 参数 | 值 |
|------|------|
| 设备类型 | 燃气表 (Gas) - 03h |
| 制造商 | ELS (93h 15h) |
| 仪表识别号 | 12345678 |
| 版本 | 51 (33h) |
| 主密钥 | 00 01 02 03 04 05 06 07 08 09 0A 0B 0C 0D 0E 0F |
| 消息计数器 | 2739 (B3 0A 00 00, LSB first) |

**派生密钥 (KDF-A)**：
```
Kenc = CMAC(MK, DC=00h || C || ID || 07h×7) = EC CF 39 D4 75 D7 30 B8 28 4F DF DC 19 95 D5 2F
Kmac = CMAC(MK, DC=01h || C || ID || 07h×7) = C9 CD 19 FF 5A 9A AD 5A 6B BD A1 3B D2 C4 C7 AD
IV = 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 (静态)
```

**AFL 字段解析**：
```
FCL = 00h 2Ch (LSB first)
  位0-7: FID=0 (不分段)
  位14: MF=0 (最后段)
  位13: MCLP=1 (MCL存在)
  位12: MLP=0 (ML不存在)
  位11: MCRP=1 (MCR存在)
  位10: MACP=1 (MAC存在)
  位9: KIP=0 (KI不存在)

MCL = 25h
  位5: MCMP=1 (消息计数器存在)
  位3-0: AT相关

MCR = B3 0A 00 00 (消息计数器=2739)
MAC = A0 85 18 CC B0 22 C5 FD (8字节, 由MCL.AT定义)
```

**TPL 字段**：
```
CF = 10h 07h (Mode 7, 1个加密块, Z=0无TPL计数器)
CFE = 10h (KeyID=0, KDF=A, V=0无版本)
```

**帧结构 (不分段)**：
```
┌─────────┬────────────────────────────────────────────────┐
│ DLL     │ 68h 31h 31h 68h 08h 03h                        │
├─────────┼────────────────────────────────────────────────┤
│ AFL     │ AFLL=90h (15字节)                              │
│         │ FCL=00h 2Ch                                    │
│         │ MCL=25h                                        │
│         │ MCR=B3 0A 00 00                                │
│         │ MAC=8字节                                       │
├─────────┼────────────────────────────────────────────────┤
│ TPL     │ CI=72h, 地址字段...                            │
│         │ CF=10h 07h (Mode 7, Z=0)                       │
│         │ CFE=10h                                        │
├─────────┼────────────────────────────────────────────────┤
│ 加密块  │ AES-Verify: 2F 2F                              │
│         │ 加密应用数据...                                │
├─────────┼────────────────────────────────────────────────┤
│ DLL尾   │ Checksum, Stop: 16h                           │
└─────────┴────────────────────────────────────────────────┘
```

**关键确认**：
- Z=0: TPL 无 counter field，由 AFL.MCR 提供消息计数器
- 不分段时只有 1 个 MAC（由 MCL.AT 定义）
- FCL.MCRP=1 和 MCL.MCMP=1 同时设置，指向同一 AFL.MCR 字段

## 十二、M-Bus 安装与注册

### 无线 M-Bus 安装流程 (EN 13757-4 Annex E)

**三种安装方案**：

| 方案 | 描述 |
|------|------|
| 方案 1 | 仪表发送 SND-IR，主站响应 CNF-IR 完成注册 |
| 方案 2 | 仪表预配置主站地址，发送 SND-NR 通知 |
| 方案 3 | 主站使用通配符广播搜索仪表 |

**SND-IR/CNF-IR 注册流程**：
```
仪表 → SND-IR (安装请求)
  包含: 制造商、识别号、版本、设备类型
  周期发送，等待响应
        ↓
主站 → CNF-IR (安装确认)
  匹配绑定参数 → 响应确认
        ↓
仪表收到 CNF-IR → 注册完成 → 进入正常通信模式
```

```
主站发送广播帧 (主地址 = 255, 次地址 = 255)
        ↓
所有仪表响应，携带自己的详细信息
        ↓
主站从中筛选目标仪表
        ↓
建立地址映射表
```

### set_encryption_key 方法 (Blue Book 4.8.3.3.7)

```
方法签名: set_encryption_key(data)
参数类型: octet-string (encryption_key)
参数内容: 16字节 AES-128 密钥

功能：
- 设置 M-Bus client 的加密密钥
- 启用加密通信

关键说明：
- 参数仅包含密钥，不含 Security Mode 参数
- 可传入空字符串禁用加密
- 安装子表后默认密钥为空（加密禁用）
```

### Security Mode 决定机制

**Security Mode 由仪表固件预设决定，不可通过协议动态配置。**

| 来源 | 说明 |
|------|------|
| EN 13757-7 SITP | 密钥传输结构不含 Security Mode 参数 |
| DLMS/COSEM | set_encryption_key 参数仅为密钥 |
| GCP 6.11.3.2.1 | "每个特定的 M-Bus 子表仅支持一种加密模式" |

**仪表支持的加密模式组合**：

| 仪表类型 | 支持的模式 | 可切换性 |
|------|------|------|
| 无加密仪表 | 仅 Mode 0 | 无法启用加密 |
| Mode 5 仪表 | Mode 0 + Mode 5 | 可在两者间切换 |
| Mode 7 仪表 | Mode 0 + Mode 7 | 可在两者间切换 |

**加密启用/禁用切换**：

| 操作 | 效果 |
|------|------|
| set_encryption_key("") | 禁用加密 → Mode 0 |
| set_encryption_key(16B密钥) | 启用加密 → Mode 5 或 7（取决于仪表固件） |

**encryption_key_status 状态转换 (Blue Book Annex B)**：

```
状态转换流程：

State (0)                    State (1)                    State (2)
no encryption key    →→→    encryption_key set    →→→    encryption_key transferred
                       set_encryption_key()          transfer_key() 或
                                                     transfer_security_information()

State (2)                    State (3)                    State (4)
encryption_key transferred  →→→  encryption_key set  →→→  encryption_key in use
                       set_encryption_key()          加密帧成功交换

四种安装场景：
a) 仪表预置密钥不可更换：直接进入 State (2)
b) 仪表预置密钥可更换：需传输新密钥后激活
c) 仪表无预置密钥可设置：需设置并传输密钥
d) 仪表不使用加密：保持 State (0)
```

**如何确定仪表使用的 Security Mode**：

```
流程：
1. ACESM 设置密钥 → 不知道模式
2. 传输密钥到仪表 → 不知道模式
3. 收到仪表响应帧 → 从 CF.MMMMM 解析 → 知道模式
4. configuration 属性更新 → 存储解析结果

Security Mode 信息来源：
- 仪表发送的每个帧携带 CF 字段
- CF.MMMMM 位（位8-12）指示 Security Mode
- ACESM 从帧解析，不是从配置设置
```

## 十四、关键设计要点

### 端到端安全 vs 链路安全

| 层级 | 保护范围 | 特点 |
|------|------|------|
| AFL/TPL MAC | 传输段 | 中间节点会剥离重新封装 |
| APL MAC | 端到端 | 随应用数据传输，不被剥离 |

### 短/长 TPL Header 选择

| 条件 | Header 类型 |
|------|------|
| 无线 + 仪表直接通信 | 短 Header |
| 无线 + 网关代理 | 长 Header |
| 有线 M-Bus | 长 Header |

### 链路层地址 vs 应用地址

| 场景 | 链路层地址 | 应用地址 |
|------|------|------|
| 仪表直接通信 | 仪表地址 | 相同（可省略） |
| 网关代理 | 网关地址 | 仪表地址（需携带） |

### 认证标签长度选择

| 应用类型 | 推荐 MAC 长度 |
|------|------|
| 消费值传输 | 4-8 字节 |
| 开关命令 | 12-16 字节 |
| 软件更新 | 16 字节 |
| 密钥传输 | 16 字节 |
