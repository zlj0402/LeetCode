## 二、链路层 (DLL)

### 有线 M-Bus (EN 13757-2)

**帧结构 (FT1.2)**：
```
┌─────┬─────┬─────┬─────┬─────┬─────┐
│Start│ L   │ C   │ A   │Data │CS   │End  │
│ 10h │     │     │主地址│     │     │16h  │
└─────┴─────┴─────┴─────┴─────┴─────┘
- 波特率: 2400 b/s, E,8,1
- 地址: 1字节主地址 + 1字节次地址
- 不包含完整仪表地址信息（制造商、识别号等）
```

**地址结构**：
- 主地址: 0-250（设备编号），255 = 广播/通配符
- 次地址: 用于选择功能

### 无线 M-Bus (EN 13757-4)

**帧结构 (FT3)**：
```
┌─────┬─────┬─────┬─────┬─────┬─────┐
│Preamble│L   │C   │Addr │Data │CRC  │
│        │     │     │8字节│     │     │
└─────┴─────┴─────┴─────┴─────┴─────┘
- 地址包含完整信息: 制造商(2B) + 识别号(4B) + 版本(1B) + 设备类型(1B)
```

**传输模式**：
| 模式 | 方向 | 频率 | 特点 |
|------|------|------|------|
| C1 | 单向 | 868 MHz | 紧凑格式，快速传输 |
| C2 | 双向 | 868 MHz | 支持确认和命令 |
| T1 | 单向 | 868 MHz | 长格式，更多数据 |
| T2 | 双向 | 868 MHz | 支持确认和命令 |
| S1/S1-m | 单向 | 868 MHz | 慢速模式，节能 |
| R2 | 双向 | 868 MHz | 中继模式 |

## 三、传输层 (TPL)

### TPL Header 类型

| 类型 | 长度 | 使用场景 |
|------|------|------|
| **无 Header** | 0 字节 | 简单传输，无安全需求 |
| **短 Header** | 4 字节 | 无线 M-Bus，仪表直接通信 |
| **长 Header** | 12 字节 | 有线 M-Bus / 网关场景 |

### 短 TPL Header 结构 (表11)

```
┌─────────┬─────────┬───────────────────┬──────────────┐
│访问号   │状态     │配置字段 (CF)       │CF扩展 (可选) │
│ 1字节   │ 1字节   │ 2字节              │ 0-n字节      │
└─────────┴─────────┴───────────────────┴──────────────┘

适用条件：
1. 无线 M-Bus 系统（链路层已包含完整地址）
2. 仪表地址与链路层地址相同（仪表直接通信）
3. 不经过网关/转换器
```

### 长 TPL Header 结构 (表12)

```
┌─────────┬─────────┬─────┬─────┬─────┬─────┬─────┬─────┐
│识别号   │制造商   │版本 │设备 │访问 │状态 │CF   │CFE  │
│ 4字节   │ 2字节   │1字节│1字节│1字节│1字节│2字节│0-n │
└─────────┴─────────┴─────┴─────┴─────┴─────┴─────┴─────┘

适用场景：
1. 有线 M-Bus（链路层无完整地址）
2. 网关/转换器代理通信（仪表地址 ≠ 链路层地址）
3. 需要明确标识数据归属仪表
```

### CI 字段 (控制信息)

| CI 值 | 含义 |
|------|------|
| 7Ah | 短 TPL header，无加密 |
| 72h | 长 TPL header，无加密 |
| 5Bh | 长 TPL header，应用层响应 |
| 8Ah | 短 TPL header，安全模式 5 |
| ... | 其他安全模式对应值 |

## 四、地址系统

### 识别号 (Identification Number)

- 编码: 8 位 BCD 紧缩数字 (4 字节)
- 范围: 00000001 ~ 99999999
- 类型:
  - **制造编号**: 出厂预设，唯一，通常不可更改
  - **可更改编号**: 可按系统集成需求修改（需另有固定制造编号）
- **通配符**: Fh 位置可替换 0-9，用于组寻址

示例: `112233FF` 匹配 `11223300` ~ `11223399`

### 制造商 ID (Manufacturer ID)

- 编码: 2 字节无符号二进制
- 计算公式: `[ASCII(1st)-64]×1024 + [ASCII(2nd)-64]×32 + [ASCII(3rd)-64]`
- 管理: FLAG 协会 (http://www.dlms.com/flag)
- **通配符**: FFFFh 匹配所有制造商

### 版本 (Version)

- 编码: 1 字节
- 用途: 设备代数/版本标识
- **通配符**: FFh 匹配所有版本

## 五、访问号 (Access Number)

### 功能

1. **检测新数据**: 通过访问号递增判断数据报是否为新数据
2. **检测重复**: 相同访问号的数据报被视为重复，应忽略
3. **检测频繁读出**: (有线M-Bus) 用户可通过访问号变化频率发现异常抄表

### 规则

**仪表发起数据报**：
- 每次传输前或后递增 (模 256)
- 上电后随机初始值 (0-255)
- 响应数据报重复接收到的伙伴访问号
- 不应可重置

**伙伴发起数据报**：
- 300s 内或频繁访问周期内使用不同值
- 简单递增推荐
- 响应时重复接收到的仪表访问号
- 上电后建议初始值 0

### 检测规则

```
接收方处理：
- 访问号 = 上次接收值 → 重复数据报 → 忽略
- 访问号 ≠ 上次接收值 → 新数据报 → 处理
```

### 仪表消息状态 (表14-15)

| 位 | 含义(置位) | 含义(未置位) |
|------|------|------|
| 7-5 | 制造商特定 | 制造商特定 |
| 4 | 暂时错误 | 无暂时错误 |
| 3 | 永久错误 | 无永久错误 |
| 2 | 电量低 | 电量正常 |
| 1-0 | 应用状态 | 见表15 |

**应用状态编码 (位1-0)**：
| 值 | 含义 |
|------|------|
| 00 | 无错误 |
| 01 | 应用忙 |
| 10 | 应用错误 |
| 11 | 异常条件/告警 |

### 错误类型说明

- **电量低**: 外部电源中断或电池寿命结束
- **永久错误**: 致命设备错误，需要服务干预
- **暂时错误**: 轻微错误，可能自动恢复
- **应用错误**: 命令解释/执行失败
- **异常条件**: 正常工作时检测异常行为（如水表持续流动）

## 八、消息计数器

### 计数器类型

| 计数器 | 用途 | 管理方 |
|------|------|------|
| CM | 仪表传输计数器 | 仪表 |
| CCP | 伙伴传输计数器 | 通信伙伴 |
| C'CP | 伙伴计数器未验证副本 | 仪表接收存储 |
| C'M | 仪表计数器未验证副本 | 伙伴接收存储 |
| C"M | 仪表计数器已验证副本 | 伙伴永久存储 |

### CM 规则 (仪表传输)

- 初始值: 0
- 每次生成认证/加密消息前递增 1
- **在未加密部分传输**（用于重放检测）
- 达到最大值回绕 → 建议在回绕前更换密钥
- 限制性系统不允许回绕 → 密钥更换时重置

### CCP 规则 (伙伴传输)

- 每仪表初始值: 0
- 生成认证/加密消息前递增
- **增量不超过 100**
- 可在收到仪表消息时设置
- 达到最大值回绕到 0-99

### 接收验证

**仪表接收伙伴消息**：
- 接收值不在 CM+1 ~ CM+100 范围 → 丢弃
- 有效范围 → 存为 C'CP
- 认证成功且 C'CP > CM → CM = C'CP

**伙伴接收仪表消息**：
- 接收值 ≤ C"M → 丢弃（重放）
- 接收值 > C"M → 存为 C'M
- 认证成功 → C"M = C'M

### 重放检测机制

```
接收方处理流程：
1. 读取未加密部分的消息计数器
2. 与本地已验证副本比较
3. ≤ 已验证值 → 旧消息重放 → 直接丢弃（无需解密）
4. > 已验证值 → 可能新消息 → 解密和 MAC 验证
5. 验证成功 → 更新已验证副本
```

## 九、安全服务 (Chapter 9)

### 安全目标 (表46)

| 安全服务 | 目标 |
|------|------|
| **完整性** | 防止未经授权方修改信息 |
| **真实性** | 确定通信方真实身份 |
| **机密性** | 防止向未经授权方披露信息 |

### 安全机制 (表47)

| 模式 | 算法 | 操作模式 | 加密 | 认证 | 开销 | MAC 长度 |
|------|------|------|------|------|------|------|
| 0 | 无 | - | - | - | 0 | 0 |
| 2 | DES | CBC | - | - | 1-8 | 0 |
| 3 | DES | CBC | - | - | 1-8 | 0 |
| 5 | AES-128 | CBC | ✓ | - | 2-17 | 0 |
| 7 | AES-128 | CBC | ✓ | CMAC | 3-23 | 2-16 |
| 8 | AES-128 | CTR | ✓ | CMAC | 6-16 | 4+2 |
| 9 | AES-128 | GCM | ✓ | ✓ | 18-21 | 12 |
| 10 | AES-128 | CCM | ✓ | ✓ | 10-23 | 4-16 |

### 模式说明

- **模式 0**: 无安全
- **模式 2/3**: DES-CBC，已弃用
- **模式 5**: 简单加密，单持久密钥，无密钥派生
- **模式 7**: 加密+AFL认证，支持密钥派生
- **模式 8**: 紧凑方案，双 MAC 端到端完整性
- **模式 9**: 单次认证加密，与 DLMS/COSEM 兼容
- **模式 10**: 认证加密，最小开销优化

### 安全模式 5 (AES-CBC-128)

**初始化向量 IV (表48)**：
```
15-14: 制造商 LSB
13-10: 识别号 LSB
9:     版本
8-5:   设备类型
4-0:   访问号（每次传输变化）
```

特点：
- 每次传输 IV 不同（访问号变化）
- 即使数据相同，加密结果也不同
- 防止零消耗检测和重放

**解密验证**: 前 2 字节为 2Fh (AES-check)

### 安全模式 7 (AES-CBC-128)

- IV = 0（静态）
- 使用临时密钥（通过 KDF 生成）
- 需要消息计数器（TPL 或 AFL.MCR）
- 数据填充至 16 字节倍数
- 解密验证: 前 2 字节 2Fh
- 认证: 使用 AFL 的 AES-CMAC (9.3.2)

**TPL 填充**: 1-16 字节，值等于填充字节数

### AFL 认证方法

**AES-CMAC-128 (9.3.2)**：
```
输入: AFL.MCL || {AFL.KI} || {AFL.MCR} || {AFL.ML} || TPDU
输出: 16字节 MAC，可截断为 4/8/12/16字节
```

**AES-GMAC-128 (9.3.3)**：
```
IV = 固定字段(8B) + 调用字段(5B)
固定字段 = 制造商ID || 识别号 || 版本 || 设备类型
调用字段 = AFL.MCR || AFL.FID
输入: AFL.MCL || {AFL.KI} || {AFL.MCR} || {AFL.ML} || TPDU
输出: 12字节 MAC，不截断
```

## 十、安全信息传输协议 (SITP - Annex A)

### SITP 服务

| 服务 | 描述 |
|------|------|
| 传输安全信息 | 以安全方式传输密钥等安全信息 |
| 激活安全信息 | 激活先前传输的密钥 |
| 停用安全信息 | 停用先前使用的密钥 |
| 销毁安全信息 | 销毁并删除密钥 |
| 组合激活/停用 | 同时激活新密钥和停用旧密钥 |
| 生成安全信息 | 指示仪表创建密钥 |
| 获取安全信息 | 从仪表获取单个密钥信息 |
| 获取所有密钥列表 | 获取所有密钥 ID 和版本列表 |
| 获取已激活密钥列表 | 获取当前已激活的密钥列表 |
| 传输端到端安全数据 | 端到端安全传输应用数据 |

### CI 字段 (表A.1)

| CI | 类型 | 描述 |
|------|------|------|
| C3h | 命令 | SITP 命令到设备（长帧头） |

| C5h | 响应 | SITP 响应（长帧头） |

## 十一、M-Bus 安装与注册

### 无线 M-Bus 安装流程 (EN 13757-4 Annex E)

**三种安装方案**：

| 方案 | 描述 |
|------|------|
| 方案 1 | 仪表发送 SND-IR，主站响应 CNF-IR 完成注册 |
| 方案 2 | 仪表预配置主站地址，发送 SND-NR 通知 |
| 方案 3 | 主站使用通配符广播搜索仪表 |

**SND-IR/CNF-IR 注册流程**：
```
仪表 → SND-IR (安装请求)
  包含: 制造商、识别号、版本、设备类型
  周期发送，等待响应
        ↓
主站 → CNF-IR (安装确认)
  匹配绑定参数 → 响应确认
        ↓
仪表收到 CNF-IR → 注册完成 → 进入正常通信模式
```

### 有线 M-Bus Wildcard Search

```
主站发送广播帧 (主地址 = 255, 次地址 = 255)
        ↓
所有仪表响应，携带自己的详细信息
        ↓
主站从中筛选目标仪表
        ↓
建立地址映射表
```

### 仪表绑定参数

仪表绑定到 Smart Meter 的条件：
- identification_number 匹配
- manufacturer_id 匹配
- version 匹配
- device_type 匹配

### M-Bus 密钥传输 (SITP)

流程：

## 十三、关键设计要点

### 端到端安全 vs 链路安全

| 层级 | 保护范围 | 特点 |
|------|------|------|
| AFL/TPL MAC | 传输段 | 中间节点会剥离重新封装 |
| APL MAC | 端到端 | 随应用数据传输，不被剥离 |

### 短/长 TPL Header 选择

| 条件 | Header 类型 |
|------|------|
| 无线 + 仪表直接通信 | 短 Header |
| 无线 + 网关代理 | 长 Header |
| 有线 M-Bus | 长 Header |

### 链路层地址 vs 应用地址

| 场景 | 链路层地址 | 应用地址 |
|------|------|------|
| 仪表直接通信 | 仪表地址 | 相同（可省略） |
| 网关代理 | 网关地址 | 仪表地址（需携带） |

### 认证标签长度选择

| 应用类型 | 推荐 MAC 长度 |
|------|------|
| 消费值传输 | 4-8 字节 |
| 开关命令 | 12-16 字节 |
| 软件更新 | 16 字节 |
| 密钥传输 | 16 字节 |
