# 无线 M-Bus 安装与注册程序

## 三种安装方案

### Solution a) - 自动配对（无预定义）

- 仪表广播信号，通信伙伴检测范围内的仪表
- **需要在线连接**到 Meter Site Operator 后台系统
- 后台系统决定将仪表分配给哪个通信伙伴
- 适用场景：新安装、大规模部署

### Solution b) - 操作员预定义

- 安装前将仪表参数（次级地址）预加载到通信伙伴
- 可通过在线连接或预配置完成
- 无需现场发现过程
- 适用场景：系统已规划好配对关系

### Solution c) - 服务技术人员现场配对

- **不需要在线连接**
- 使用 SND-IR (Installation Request) 手动注册
- 技术人员在现场直接完成配对
- 适用场景：小规模、离线安装

## 有仪表支持的注册 (D.2)

### SND-IR (Send Installation Request)

- 由按钮或移动设备命令触发
- 仪表发送多条安装消息（短间隔）
- 定义于 EN 13757-4

### RF-Link Feedback

- 通信伙伴响应 SND-IR，发送 SND-NKE
- 包含仪表链路地址和伙伴链路层地址
- 在双向电池仪表的接收窗口外传输（避免干扰）
- 可包含 RF 接收电平 (RSSI)
- 移动接收工具可显示/存储，帮助判断链路质量

### CNF-IR (Confirm Installation Request)

- 通信伙伴在 Registration Mode 下响应
- 在双向仪表的小型接收窗口传输
- 确认注册成功，仪表停止发送安装消息

## 无仪表支持的注册 (D.3)

### 特点

- 仪表无需特殊安装功能
- 安装后直接开始传输（或始终传输）
- 通信伙伴自动检测未知仪表消息

### 工作流程

1. 通信伙伴收到未知仪表消息
2. 记录仪表链路地址 + 接收时间到特殊列表
3. 后台系统通知或定期查询
4. 后台系统分配仪表到特定通信伙伴
5. 密钥下载到通信伙伴（需高层隐私链路）

### 优势

- 安装流程简单快速
- 无需进入公寓即可更换通信伙伴
- 支持两种安装顺序（先仪表或先伙伴）

### 缺点

- 安装时无链路质量反馈给技术人员
