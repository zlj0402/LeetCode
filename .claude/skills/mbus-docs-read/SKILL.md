name: mbus 协议文档阅读方式
description: 当用户要求阅读 mbus 协议文档时，结合文档理解某个协议点，或者是确认某个协议点时，请参考此规范；
---

# 指令

1. 对于每一个协议文档，建立一个专有的该文档 memeory，已有 memory 按照后面的指令进行；
2. 首先查看有没有该协议文档的 memory 有没有，再去看看有没有问题涉及的章节，每当阅读到该文档 memory 当中没有记录的章节时，以源文档的内容(原封不动的)，按章节顺序更新到该文档的 memory；
  2.1 遇到表格类的合并单元格，请仔细校对该单元格的范围，不要弄错了；
3. 讨论到的理解，有了确切的结果之后，在相应的小结之后，以补充的形式更新到协议的 memory 当中；

## 参考示例

- 问题：en13757-7:2018 中 security mode 5 是 AES-128 CTR 还是 AES-128-CBC?
  - 正确的做法：
    1. 比如在 7.5.8 Configuration field 章节中看到各个 security mode 的 Characteristics 表格，以及具体的参考章节；就把 7.5.8 章节和具体的参考章节 9.4.4，将这两个小节的内容，原封不动的协议内容，按照章节标题的形式，更新到该协议的 memory 当中；
      1.1 协议内容已经有了的话，就不更新，只将没有的更新；
    2. 并将需要理解的问题，以 Note 的形式加到相应小节内容后；
